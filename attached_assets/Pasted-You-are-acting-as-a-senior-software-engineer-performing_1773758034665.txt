You are acting as a senior software engineer performing a full technical audit of this project.

This project previously worked correctly, but after several AI-assisted updates, multiple parts of the system are now malfunctioning. The exact failures are unclear, so the goal is to diagnose the entire system and produce a detailed technical report.

Your task is to perform a complete READ-ONLY system audit.

IMPORTANT RULES

- DO NOT modify any code
- DO NOT modify the database
- DO NOT create or delete files
- DO NOT implement fixes
- Your role is diagnosis and reporting only

--------------------------------------------------

PHASE 1 — PROJECT ARCHITECTURE SCAN

Analyze the entire project structure and document:

- Directory structure
- Frontend framework and architecture
- Backend integrations
- Configuration files
- Dependency management
- Build configuration

Explain:

- How the system is supposed to work
- How the components interact
- The data flow between frontend and backend

--------------------------------------------------

PHASE 2 — CODEBASE INTEGRITY ANALYSIS

Scan the entire codebase and detect:

- Broken imports
- Missing files
- Invalid module references
- Dependency conflicts
- Unused or orphaned code
- Incorrect API usage
- Misconfigured environment variables

Report:

- Files with errors
- Code that references non-existent functions
- Logic that may break runtime behavior

--------------------------------------------------

PHASE 3 — SUPABASE DEEP INSPECTION

Use the provided service role key to inspect the entire Supabase project.

Audit the following:

Database
- Tables
- Columns
- Relationships
- Constraints
- Indexes

Security
- Row Level Security (RLS) policies
- Permissions

Backend Logic
- Edge Functions
- Triggers
- Stored procedures (if any)

Platform Setup
- Authentication configuration
- Storage buckets
- API usage

Check for:

- Schema mismatches with the codebase
- Queries referencing missing tables or columns
- RLS policies blocking expected operations
- Edge functions failing or misconfigured

--------------------------------------------------

PHASE 4 — INTEGRATION VALIDATION

Trace how the frontend communicates with Supabase.

Analyze:

- Supabase client configuration
- API calls
- Query structure
- Data fetching logic
- Authentication flow

Identify:

- Broken queries
- Incorrect table references
- Incorrect API usage
- Authentication failures
- Permission errors

--------------------------------------------------

PHASE 5 — RUNTIME FAILURE DETECTION

Simulate the expected system behavior and identify:

- Where errors would occur
- Which components would fail
- Which functions cannot execute successfully

Document:

- Critical failures
- High-risk areas
- Performance or security concerns

--------------------------------------------------

PHASE 6 — ENVIRONMENT & CONFIGURATION AUDIT

Inspect:

- Environment variables
- Build configuration
- Deployment settings
- Supabase project connection settings

Identify:

- Missing environment variables
- Misconfigured keys
- Incorrect project URLs
- Invalid deployment configuration

--------------------------------------------------

PHASE 7 — REQUIRED MANUAL ACTIONS

If any issues require manual intervention, clearly list them.

Examples may include:

- Changes needed in the Supabase dashboard
- Environment variables that must be added
- Permissions that must be adjusted
- Storage configuration fixes

--------------------------------------------------

PHASE 8 — FINAL TECHNICAL REPORT

Produce a structured diagnostic report containing:

1. SYSTEM OVERVIEW
Explain how the project is structured and how it should work.

2. WHAT IS WORKING
List the parts of the system currently functioning correctly.

3. DETECTED ISSUES
Provide a detailed list of all discovered problems.

4. ROOT CAUSE ANALYSIS
Explain the likely cause of each issue.

5. RECOMMENDED FIX STRATEGY
Explain the best approach to fix the system.

6. MANUAL STEPS REQUIRED
List any actions that must be performed by the project owner.

--------------------------------------------------

OUTPUT REQUIREMENTS

Your output must be clear, structured, and highly detailed so that another AI or developer could use the report to repair the system efficiently.

The goal is to produce a complete technical diagnosis of the project.
